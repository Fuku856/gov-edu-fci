# Vercel デプロイガイド

このガイドでは、生徒会活動資金見える化プロジェクトを Vercel にデプロイする手順を詳しく説明します。

## 📋 事前準備

### 1. GitHub アカウントの準備
- [GitHub](https://github.com) にアカウントを作成（まだの場合）
- リポジトリを作成しておく

### 2. Vercel アカウントの準備
- [Vercel](https://vercel.com) にアクセス
- 「Sign Up」から GitHub アカウントで登録（無料プランでOK）

## 🚀 デプロイ手順

### ステップ 1: GitHub にプッシュ

1. **Git リポジトリの初期化**（まだの場合）
   ```bash
   git init
   git add .
   git commit -m "Initial commit: 収支システムを追加"
   ```

2. **GitHub にリポジトリを作成**
   - GitHub で新しいリポジトリを作成（例: `gov-edu-fci`）
   - リポジトリ作成後、表示されるコマンドを実行：
   ```bash
   git remote add origin https://github.com/あなたのユーザー名/gov-edu-fci.git
   git branch -M main
   git push -u origin main
   ```

### ステップ 2: Vercel でプロジェクトをインポート

1. **Vercel にログイン**
   - [vercel.com](https://vercel.com) にアクセス
   - GitHub アカウントでログイン

2. **新しいプロジェクトを作成**
   - ダッシュボードから「Add New...」→「Project」をクリック
   - または「Import Project」をクリック

3. **GitHub リポジトリを選択**
   - 表示されたリポジトリ一覧から `gov-edu-fci` を選択
   - 「Import」をクリック

### ステップ 3: プロジェクト設定

Vercel の設定画面で以下を確認・設定：

#### Framework Preset（フレームワーク設定）
- **「Other」を選択** または **「Other」のまま**
- 静的サイトなので特別なフレームワークは不要

#### Root Directory（ルートディレクトリ）
- **空欄のまま**（プロジェクトルートがそのまま使われます）

#### Build and Output Settings（ビルド設定）
- **Build Command**: 空欄のまま（静的サイトなのでビルド不要）
- **Output Directory**: 空欄のまま（ルートディレクトリがそのまま出力されます）

#### Environment Variables（環境変数）
- 今回は不要（ローカルストレージベースなので）

### ステップ 4: デプロイ実行

1. **「Deploy」ボタンをクリック**
   - デプロイが開始されます（約30秒〜1分）

2. **デプロイ完了を待つ**
   - ログが表示され、完了すると「Success!」と表示されます

3. **デプロイされたURLを確認**
   - 例: `https://gov-edu-fci.vercel.app`
   - このURLが自動的に生成されます

## 🔧 カスタムドメインの設定（オプション）

独自ドメインを使用したい場合：

1. **Vercel ダッシュボードでプロジェクトを開く**
2. **「Settings」→「Domains」を選択**
3. **ドメイン名を入力して「Add」**
4. **DNS設定を確認**
   - Vercel が表示するDNS設定を、ドメイン管理画面で設定

## 📝 よくある質問

### Q: データはどこに保存されますか？
A: 現在はブラウザのローカルストレージに保存されます。つまり、デバイスごとにデータが保存されます。

### Q: 複数のデバイスでデータを共有したい
A: データベース（Supabase、PlanetScale など）を追加する必要があります。現在の構成では、各デバイスで独立してデータが保存されます。

### Q: デプロイ後に変更を反映するには？
A: GitHub にプッシュすると、Vercel が自動的に再デプロイします（自動デプロイが有効な場合）。

### Q: ビルドエラーが出る
A: 静的サイトなので、通常はビルドエラーは出ません。出た場合は、Vercel のログを確認してください。

## 📞 トラブルシューティング

### デプロイが失敗する場合
- `package.json` が正しく作成されているか確認
- `vercel.json` の構文が正しいか確認
- Vercel のログを確認

### サイトが表示されない場合
- デプロイが完了しているか確認
- ブラウザのキャッシュをクリア
- 別のブラウザで試す

### データが消える場合
- ローカルストレージはブラウザの設定で削除されることがあります
- データベースを追加することを推奨します
